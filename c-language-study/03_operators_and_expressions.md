# 3일차 - 연산자와 표현식

## 오늘의 목표

- 우선순위와 결합 방향을 이해한다.
- 증감 연산자와 대입 연산자의 동작을 구분한다.

- `sizeof`, 조건 연산자, 쉼표 연산자까지 폭넓게 익힌다.

## 표현식이란?

표현식(expression)은 `값을 만들어내는 코드 조각`입니다.

예:

```c
10
a + b
x * y + 3
func(5)
```

문장(statement)은 실행 단위이고, 표현식은 값 중심이라는 차이가 있습니다.

```c
a = b + 3;   // 하나의 문장 안에 여러 표현식이 있음
```

## 산술 연산자

| 연산자 | 의미 |
| --- | --- |
| `+` | 덧셈 |
| `-` | 뺄셈 |
| `*` | 곱셈 |
| `/` | 나눗셈 |
| `%` | 나머지 |

```c
int a = 10;
int b = 3;

printf("%d\n", a + b); // 13
printf("%d\n", a - b); // 7
printf("%d\n", a * b); // 30
printf("%d\n", a / b); // 3
printf("%d\n", a % b); // 1
```

## 부호 연산자

```c
int x = 5;
int y = -x; // -5
int z = +x; // 5
```

## 대입 연산자

```c
int a = 10;
a = 20;
```

오른쪽 값을 계산한 후 왼쪽 변수에 저장합니다.

### 복합 대입 연산자

```c
a += 3;  // a = a + 3;
a -= 2;  // a = a - 2;
a *= 4;  // a = a * 4;
a /= 5;  // a = a / 5;
a %= 2;  // a = a % 2;
```

비트 연산에도 복합 대입이 있습니다.

```c
x &= mask;
x |= flag;
x ^= bit;
x <<= 1;
x >>= 1;
```

## 비교 연산자

| 연산자 | 의미 |
| --- | --- |
| `==` | 같다 |
| `!=` | 다르다 |
| `>` | 크다 |
| `<` | 작다 |
| `>=` | 크거나 같다 |
| `<=` | 작거나 같다 |

비교 결과는 참이면 0이 아닌 값, 거짓이면 0으로 취급됩니다.

```c
int a = 10;
int b = 20;
printf("%d\n", a < b);  // 1
printf("%d\n", a == b); // 0
```

주의:

```c
if (a = 10)
```

이 코드는 비교가 아니라 대입입니다. 매우 흔한 실수입니다.

## 논리 연산자

| 연산자 | 의미 |
| --- | --- |
| `&&` | 논리 AND |
| `||` | 논리 OR |
| `!` | 논리 NOT |

```c
int age = 20;

if (age >= 19 && age <= 29)
{
    printf("20대입니다.\n");
}
```

### 단축 평가(short-circuit evaluation)

```c
if (ptr != NULL && *ptr > 0)
{
    ...
}
```

첫 조건이 거짓이면 두 번째는 평가하지 않습니다. 그래서 널 포인터 검사에서 매우 중요합니다.

## 증감 연산자

| 연산자 | 의미 |
| --- | --- |
| `++x` | 먼저 증가 후 사용 |
| `x++` | 먼저 사용 후 증가 |
| `--x` | 먼저 감소 후 사용 |
| `x--` | 먼저 사용 후 감소 |

```c
int x = 5;
int a = ++x; // x = 6, a = 6
```

```c
int y = 5;
int b = y++; // b = 5, y = 6
```

복잡한 식에서 남용하면 읽기 어려워집니다.

```c
arr[i++] = ++j;
```

이런 식은 초보자에게 매우 헷갈리므로 피하는 편이 좋습니다.

## 비트 연산자

| 연산자 | 의미 |
| --- | --- |
| `&` | 비트 AND |
| `|` | 비트 OR |
| `^` | 비트 XOR |
| `~` | 비트 NOT |
| `<<` | 왼쪽 시프트 |
| `>>` | 오른쪽 시프트 |

예:

```c
unsigned int a = 6;  // 0110
unsigned int b = 3;  // 0011

printf("%u\n", a & b); // 2  -> 0010
printf("%u\n", a | b); // 7  -> 0111
printf("%u\n", a ^ b); // 5  -> 0101
```

### 비트 마스크 예제

```c
#include <stdio.h>

int main(void)
{
    unsigned int flags = 0;

    flags |= 1u << 0; // 0번 비트 켜기
    flags |= 1u << 2; // 2번 비트 켜기

    if (flags & (1u << 2))
    {
        printf("2번 비트가 켜져 있습니다.\n");
    }

    flags &= ~(1u << 0); // 0번 비트 끄기

    printf("flags = %u\n", flags);
    return 0;
}
```

## 주소 연산자와 역참조 연산자

포인터에서 자세히 다루지만 미리 개념만 보면:

| 연산자 | 의미 |
| --- | --- |
| `&x` | x의 주소 |
| `*p` | p가 가리키는 값 |

## `sizeof`

```c
printf("%zu\n", sizeof(int));
printf("%zu\n", sizeof a);
```

`sizeof`는 함수가 아니라 연산자입니다.

## 형 변환 연산자

```c
double x = 10.7;
int y = (int)x;
```

## 조건 연산자 `?:`

```c
int score = 85;
char grade = (score >= 80) ? 'P' : 'F';
```

간단한 선택에는 유용하지만, 너무 복잡하게 쓰면 가독성이 나빠집니다.

## 쉼표 연산자

왼쪽 표현식을 먼저 평가하고, 전체 값은 오른쪽 표현식의 값이 됩니다.

```c
int x = (printf("Hello\n"), 10);
```

실무에서는 자주 쓰지 않지만 `for`문에서 볼 수 있습니다.

```c
for (i = 0, j = 10; i < j; ++i, --j)
{
    ...
}
```

## 우선순위와 결합 방향

정확한 표 전체를 외우는 것보다 다음 원칙을 먼저 기억하세요.

1. 괄호가 가장 강력하다.
2. 단항 연산자가 보통 강하다.
3. 산술 연산이 비교 연산보다 먼저다.
4. 비교 연산이 논리 연산보다 먼저다.
5. 대입은 뒤쪽에서 실행된다.

예:

```c
int x = 2 + 3 * 4; // 14
```

```c
int y = (2 + 3) * 4; // 20
```

### 결합 방향 예시

```c
int a, b, c;
a = b = c = 10;
```

대입은 오른쪽에서 왼쪽으로 결합합니다.

## 예제: 홀수/짝수 판별

```c
#include <stdio.h>

int main(void)
{
    int n;
    scanf("%d", &n);

    if (n % 2 == 0)
    {
        printf("짝수\n");
    }
    else
    {
        printf("홀수\n");
    }

    return 0;
}
```

## 예제: 윤년 판별

```c
#include <stdio.h>

int main(void)
{
    int year;
    scanf("%d", &year);

    if ((year % 4 == 0 && year % 100 != 0) || (year % 400 == 0))
    {
        printf("윤년입니다.\n");
    }
    else
    {
        printf("윤년이 아닙니다.\n");
    }

    return 0;
}
```

## 실수 포인트

- `=`와 `==`를 헷갈림
- `&&`와 `&`, `||`와 `|`를 혼동함
- 정수 나눗셈을 실수 나눗셈처럼 생각함
- 증감 연산자를 복잡한 식에서 무리하게 사용함
- 괄호 없이 우선순위를 추측함

## 오늘 정리

- 표현식은 값을 만들어내는 코드입니다.
- 산술, 비교, 논리, 비트, 대입 연산자는 서로 다릅니다.
- 연산자 우선순위는 외우기보다 `괄호`로 명확히 쓰는 습관이 중요합니다.
