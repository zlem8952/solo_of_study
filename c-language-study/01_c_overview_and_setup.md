# 1일차 - C 언어 개요와 시작하기

## 오늘의 목표

- C 언어가 어떤 언어인지 이해한다.
- 가장 기본적인 프로그램 구조를 익힌다.
- 소스 코드가 컴파일되어 실행 파일이 되는 흐름을 안다.
- `main` 함수, 문장, 블록, 주석의 의미를 이해한다.

## C 언어란?

C는 시스템 프로그래밍에 매우 강한 언어입니다. 운영체제, 임베디드, 게임 엔진, 컴파일러, 라이브러리, 네트워크 프로그램 등에서 오랫동안 사용되어 왔습니다.

### C 언어의 특징

- 실행 속도가 빠릅니다.
- 메모리를 직접 다룰 수 있습니다.
- 하드웨어와 가까운 제어가 가능합니다.
- 표준 라이브러리가 비교적 작고 명확합니다.
- 반대로, 안전장치가 적어서 실수도 직접 관리해야 합니다.

## C 프로그램의 최소 구조

```c
#include <stdio.h>

int main(void)
{
    printf("Hello, C!\n");
    return 0;
}
```

## 코드 해설

### `#include <stdio.h>`

- `stdio.h`는 표준 입출력 헤더입니다.
- `printf` 같은 함수를 사용하려면 선언이 필요하고, 그 선언을 헤더가 제공합니다.

### `int main(void)`

- 프로그램의 시작점입니다.
- `int`는 반환형입니다.
- `main`은 운영체제에 종료 상태 코드를 돌려줍니다.
- 보통 성공이면 `0`, 실패면 `0`이 아닌 값을 반환합니다.

### `{ ... }`

- 중괄호는 `블록(block)`입니다.
- 여러 문장을 하나의 묶음으로 만듭니다.

### `printf("Hello, C!\n");`

- 문자열을 화면에 출력합니다.
- `\n`은 줄바꿈 문자입니다.
- C에서 문장은 대부분 세미콜론 `;`으로 끝납니다.

### `return 0;`

- 프로그램이 정상 종료되었음을 의미합니다.

## 들여쓰기와 공백

C는 들여쓰기가 문법은 아니지만, 코드 가독성에 매우 중요합니다.

```c
int main(void)
{
    int a = 10;
    int b = 20;
    printf("%d\n", a + b);
    return 0;
}
```

추천 습관:

- 중괄호는 눈에 잘 보이게 정렬합니다.
- 한 줄에 너무 많은 코드를 넣지 않습니다.
- 의미 단위로 빈 줄을 넣습니다.

## 주석

### 한 줄 주석

```c
// 이것은 한 줄 주석입니다.
```

### 여러 줄 주석

```c
/*
   이것은
   여러 줄 주석입니다.
*/
```

주의:

- 주석 안에 또 `/* ... */`를 중첩할 수는 없습니다.
- 주석은 설명을 위한 것이지, 나쁜 코드의 변명용이 아닙니다.

## C 프로그램이 실행되기까지

소스 파일 `main.c`가 있다고 가정하면 다음 단계를 거칩니다.

1. 전처리(Preprocessing)
2. 컴파일(Compilation)
3. 어셈블(Assembling)
4. 링크(Linking)

### 1. 전처리

- `#include`, `#define`, `#if` 같은 전처리 지시문을 처리합니다.
- 헤더 파일이 끼워 넣어집니다.

### 2. 컴파일

- C 코드를 어셈블리 코드로 변환합니다.
- 문법 오류가 여기서 많이 잡힙니다.

### 3. 어셈블

- 어셈블리 코드를 기계어 목적 파일로 바꿉니다.

### 4. 링크

- 여러 목적 파일과 라이브러리를 합쳐 실행 파일을 만듭니다.

## 예제: 계산 결과 출력

```c
#include <stdio.h>

int main(void)
{
    int width = 5;
    int height = 8;
    int area = width * height;

    printf("width = %d\n", width);
    printf("height = %d\n", height);
    printf("area = %d\n", area);

    return 0;
}
```

## `printf`의 기본 서식

`printf`는 서식 문자열을 사용합니다.

| 서식 | 의미 |
| --- | --- |
| `%d` | 정수 출력 |
| `%u` | 부호 없는 정수 출력 |
| `%f` | 실수 출력 |
| `%c` | 문자 출력 |
| `%s` | 문자열 출력 |
| `%p` | 포인터 주소 출력 |

예시:

```c
#include <stdio.h>

int main(void)
{
    int age = 20;
    double score = 95.5;
    char grade = 'A';

    printf("age = %d\n", age);
    printf("score = %.1f\n", score);
    printf("grade = %c\n", grade);

    return 0;
}
```

## 식별자 이름 규칙

식별자는 변수명, 함수명, 배열명 등에 사용되는 이름입니다.

### 가능

- 영문자
- 숫자
- 밑줄 `_`

### 불가능

- 숫자로 시작
- 공백 포함
- 특수문자 대부분 사용
- 예약어 사용

예시:

```c
int count;
int student_age;
int value2;
```

잘못된 예시:

```c
int 2value;    // 숫자로 시작
int my value;  // 공백 포함
int return;    // 예약어
```

다음은 C에서 특별한 의미를 가진 예약어입니다.

```c
int, char, if, else, for, while, return, break, continue, switch, case,
struct, union, enum, typedef, const, static, extern, sizeof
```

예약어는 변수명으로 쓸 수 없습니다.

## 에러의 종류

### 문법 오류

```c
printf("Hello")
```

세미콜론이 없어서 문법 오류가 발생합니다.

### 링크 오류

함수를 선언만 해놓고 정의하지 않으면 링크 단계에서 실패할 수 있습니다.

### 실행 오류

배열 범위를 벗어나거나 잘못된 포인터를 사용하면 실행 중 문제가 생깁니다.

## 컴파일 옵션 추천

초보자일수록 경고를 많이 켜야 합니다.

```bash
gcc -std=c11 -Wall -Wextra -pedantic main.c -o main
```

옵션 설명:

- `-std=c11`: C11 표준으로 컴파일
- `-Wall`: 자주 발생하는 경고 활성화
- `-Wextra`: 추가 경고 활성화
- `-pedantic`: 표준 위반에 더 엄격

## 자주 하는 실수

- `main` 함수 괄호와 중괄호를 빠뜨림
- `printf` 끝에 세미콜론을 빼먹음
- 문자열을 작은따옴표로 쓰는 실수
- 문자 하나를 큰따옴표로 쓰는 실수

예:

```c
char ch = "A";   // 잘못된 형태
char ch = 'A';   // 올바른 형태
```

## 오늘 정리

- C 프로그램의 시작점은 `main` 함수입니다.
- 문장은 세미콜론으로 끝나는 경우가 많습니다.
- `#include`는 필요한 선언을 가져오는 전처리 지시문입니다.
- 코드 작성뿐 아니라 `컴파일 과정`을 이해해야 C가 쉬워집니다.
