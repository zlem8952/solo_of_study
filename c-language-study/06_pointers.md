# 6일차 - 포인터

## 오늘의 목표

- 포인터가 왜 필요한지 이해한다.
- 주소 연산자 `&`, 역참조 연산자 `*`를 정확히 구분한다.
- 배열과 포인터의 관계를 이해한다.
- 포인터 연산, `NULL`, `void *`, 이중 포인터의 개념을 익힌다.

## 포인터란?

포인터는 `메모리 주소를 저장하는 변수`입니다.

```c
int x = 10;
int *p = &x;
```

의미:

- `x`는 정수 10을 저장
- `&x`는 `x`의 주소
- `p`는 `x`의 주소를 저장하는 포인터
- `*p`는 `p`가 가리키는 주소에 있는 값

## 가장 중요한 두 연산자

| 기호 | 이름 | 의미 |
| --- | --- | --- |
| `&` | 주소 연산자 | 변수의 주소를 구함 |
| `*` | 역참조 연산자 | 주소가 가리키는 값을 읽거나 씀 |

예:

```c
#include <stdio.h>

int main(void)
{
    int x = 10;
    int *p = &x;

    printf("x = %d\n", x);
    printf("&x = %p\n", (void *)&x);
    printf("p = %p\n", (void *)p);
    printf("*p = %d\n", *p);

    return 0;
}
```

## 포인터 선언 읽는 법

```c
int *p;
char *cptr;
double *dptr;
```

읽는 방법:

- `p`는 `int`를 가리키는 포인터
- `cptr`은 `char`를 가리키는 포인터

## 왜 포인터가 필요한가?

1. 함수에서 원본 값을 바꾸기 위해
2. 배열과 문자열을 효율적으로 다루기 위해
3. 동적 메모리를 사용하기 위해
4. 자료구조(연결 리스트, 트리 등)를 만들기 위해

## 역참조로 값 변경

```c
int x = 10;
int *p = &x;

*p = 99;
printf("%d\n", x); // 99
```

포인터를 통해 원본 데이터를 직접 수정할 수 있습니다.

## 잘못된 포인터 사용

```c
int *p;
*p = 10; // 매우 위험
```

`p`가 어디를 가리키는지 정해지지 않았기 때문에 정의되지 않은 동작입니다.

## `NULL` 포인터

아무 곳도 가리키지 않는 포인터를 나타냅니다.

```c
int *p = NULL;
```

검사:

```c
if (p != NULL)
{
    printf("%d\n", *p);
}
```

`NULL` 포인터를 역참조하면 안 됩니다.

## 포인터와 함수

### 값 전달의 한계

```c
void change(int x)
{
    x = 100;
}
```

이 함수는 원본 변수를 바꾸지 못합니다.

```c
int a = 10;
change(a);
printf("%d\n", a); // 10
```

### 포인터를 사용한 변경

```c
void change(int *x)
{
    *x = 100;
}

int main(void)
{
    int a = 10;
    change(&a);
    printf("%d\n", a); // 100
    return 0;
}
```

## 예제: swap 함수

```c
#include <stdio.h>

void swap(int *a, int *b)
{
    int temp = *a;
    *a = *b;
    *b = temp;
}

int main(void)
{
    int x = 10;
    int y = 20;

    swap(&x, &y);

    printf("x = %d, y = %d\n", x, y);
    return 0;
}
```

## 배열과 포인터

배열 이름은 많은 상황에서 첫 번째 원소의 주소처럼 동작합니다.

```c
int arr[3] = {10, 20, 30};

printf("%d\n", arr[0]);
printf("%d\n", *arr);
printf("%d\n", *(arr + 1));
```

결과:

- `arr[0]`과 `*arr`는 같은 값
- `*(arr + 1)`은 `arr[1]`

## 포인터 산술

포인터에 1을 더하면 `자료형 크기만큼` 이동합니다.

```c
int arr[3] = {10, 20, 30};
int *p = arr;

printf("%d\n", *p);     // 10
printf("%d\n", *(p+1)); // 20
printf("%d\n", *(p+2)); // 30
```

### 왜 `p + 1`이 1바이트가 아닌가?

`p`가 `int *`이면 다음 `int` 위치로 이동해야 하기 때문입니다. 즉, 타입 정보가 포인터 이동 단위를 결정합니다.

## 포인터와 배열 인덱스의 관계

```c
arr[i] == *(arr + i)
```

이 관계는 매우 중요합니다.

## 문자열과 포인터

```c
char str[] = "Hello";
char *p = str;

printf("%c\n", *p);     // H
printf("%c\n", *(p+1)); // e
```

### 문자열 리터럴 주의

```c
char *s = "Hello";
```

문자열 리터럴은 읽기 전용 영역에 놓일 수 있으므로 수정하면 안 됩니다.

```c
s[0] = 'h'; // 위험
```

수정 가능한 문자열이 필요하면:

```c
char s[] = "Hello";
s[0] = 'h';
```

## 포인터와 `const`

### 1. 가리키는 값을 바꾸면 안 되는 포인터

```c
const int *p;
```

또는

```c
int const *p;
```

의미:

- `*p`를 통해 값을 바꿀 수 없음
- `p`가 다른 주소를 가리키게 하는 것은 가능

### 2. 포인터 자체를 바꾸면 안 되는 경우

```c
int *const p = &x;
```

의미:

- `p`는 항상 같은 주소를 가리킴
- 하지만 `*p`는 수정 가능

### 3. 둘 다 금지

```c
const int *const p = &x;
```

## `void *`

자료형이 정해지지 않은 일반 포인터입니다.

```c
void *vp;
int x = 10;
vp = &x;
```

`void *`는 직접 역참조할 수 없습니다. 먼저 적절한 타입으로 변환해야 합니다.

```c
printf("%d\n", *(int *)vp);
```

## 이중 포인터

포인터를 가리키는 포인터입니다.

```c
int x = 10;
int *p = &x;
int **pp = &p;

printf("%d\n", **pp); // 10
```

이중 포인터는 문자열 배열, 동적 2차원 구조, 함수에서 포인터 자체를 바꿀 때 자주 사용합니다.

## 함수에서 포인터 자체 변경하기

```c
void set_null(int **pp)
{
    *pp = NULL;
}
```

## 포인터 비교

```c
if (p == NULL)
{
    ...
}
```

같은 배열 안의 포인터끼리는 비교가 의미가 있지만, 전혀 관계없는 객체의 포인터 비교는 신중해야 합니다.

## 자주 만나는 위험한 개념

### 댕글링 포인터

유효하지 않은 메모리를 가리키는 포인터입니다.

```c
int *p = NULL;
{
    int x = 10;
    p = &x;
}
printf("%d\n", *p); // 위험: x는 이미 수명 종료
```

### 와일드 포인터

초기화되지 않은 포인터입니다.

```c
int *p;
```

### 널 포인터 역참조

```c
int *p = NULL;
printf("%d\n", *p); // 위험
```

## 예제: 배열 합 구하기

```c
#include <stdio.h>

int sum_array(const int *arr, int size)
{
    int sum = 0;

    for (int i = 0; i < size; i++)
    {
        sum += arr[i];
    }

    return sum;
}

int main(void)
{
    int nums[] = {1, 2, 3, 4, 5};
    printf("%d\n", sum_array(nums, 5));
    return 0;
}
```

`const int *arr`로 받은 이유:

- 함수가 배열 값을 읽기만 한다는 의도를 분명히 하기 위해
- 실수로 수정하는 것을 막기 위해

## 실수 포인트

- 포인터를 선언만 하고 바로 역참조함
- `&`와 `*`의 의미를 혼동함
- `NULL` 검사 없이 역참조함
- 배열 이름과 포인터가 완전히 같은 것이라고 오해함
- 문자열 리터럴을 수정하려고 함

## 오늘 정리

- 포인터는 주소를 저장합니다.
- `&`는 주소, `*`는 그 주소가 가리키는 값입니다.
- 배열, 문자열, 함수, 동적 메모리의 핵심에는 포인터가 있습니다.
- C를 잘하려면 포인터를 `암기`가 아니라 `메모리 그림`으로 이해해야 합니다.
